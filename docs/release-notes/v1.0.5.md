# ghcli v1.0.5

This release makes `ghcli` vault-agnostic for standalone users and OpenClaw
skill installs.

## Highlights

- Removes the KeePassXC requirement from OAuth token storage.
- Stores user OAuth tokens in a private local token file under ghcli state.
- Adds `--credentials` and `GHCLI_GOOGLE_CREDENTIALS_JSON` for explicit Google
  OAuth client credential input.
- Removes implicit repo-local `ghapi-credentials.json` discovery.
- Updates the OpenClaw skill metadata so ClawHub installs can provide Google
  OAuth client JSON through `skills.entries.google-health-local-archive.apiKey`,
  including OpenClaw SecretRef-backed values.

No archive schema or Google Health sync behavior changed in this release.
