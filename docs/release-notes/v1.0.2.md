# ghcli v1.0.2

This release tightens the ClawHub metadata for the bundled
`google-health-local-archive` skill.

## Highlights

- Restores `metadata.openclaw.requires.bins: ["ghcli"]` so the skill's declared
  requirements match the command it instructs agents to use.
- Updates the skill install stanza to
  `github.com/fdsouvenir/ghcli@v1.0.2`.
- Keeps the skill's query behavior local and read-only through
  `ghcli --json --read-only`.
- Documents the fresh-install order outside the gated skill path:
  install `ghcli`, run OAuth setup/login, then sync once before asking
  health-data questions.

## ClawScan Context

The skill depends on the separately installed `ghcli` binary. The skill itself
does not contain executable code and does not call Google Health directly.
Agents use it to query a local SQLite archive through `ghcli`; remote API access
is limited to explicit user-run `ghcli auth` and `ghcli sync` commands.
