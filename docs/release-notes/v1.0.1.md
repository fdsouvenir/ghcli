# ghcli v1.0.1

This release improves the ClawHub fresh-install path for the bundled
`google-health-local-archive` skill.

## Highlights

- Skill metadata now uses single-line JSON for OpenClaw/ClawHub compatibility.
- The skill install stanza points to `github.com/fdsouvenir/ghcli@v1.0.1`.
- The skill no longer hard-gates on `requires.bins`, so it can tell a fresh
  user how to install `ghcli` when the binary is missing.
- Fresh installs now have an explicit diagnostic flow for:
  - missing `ghcli` binary
  - missing Google OAuth credentials
  - missing KeePassXC token state
  - empty or stale local archive
- Normal health questions remain local and read-only through
  `ghcli --json --read-only`.
- Setup, login, sync, and export commands are only allowed after explicit user
  intent.

No sync schema or Google Health API behavior changed in this release.
