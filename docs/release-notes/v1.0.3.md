# ghcli v1.0.3

This release fixes issues found during a fresh ClawHub-install pass.

## Highlights

- `go install github.com/fdsouvenir/ghcli@v1.0.3` now reports `v1.0.3` from
  `ghcli version` without requiring linker flags.
- Empty query responses now return `[]` instead of `null`.
- First-run SQLite migrations are serialized so simultaneous diagnostics and
  queries do not race while creating the local archive.
- The bundled skill install stanza now points to
  `github.com/fdsouvenir/ghcli@v1.0.3`.

The skill still declares `ghcli` as a required binary for ClawHub/OpenClaw
metadata coherence and keeps normal health-data questions local and read-only.
