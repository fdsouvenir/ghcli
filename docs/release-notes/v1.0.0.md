# ghcli v1.0.0

`ghcli` is now packaged as a Google Health-first local CLI and archive.

## Highlights

- Local-first Google Health sync into SQLite.
- Explicit OAuth setup with tokens stored in KeePassXC.
- Read-only query commands suitable for agents and shell workflows.
- Historical backfill plus incremental sync.
- Systemd user timer templates for frequent polling within API limits.
- Raw API payload storage is opt-in; normalized records are stored by default.
- Bundled OpenClaw skill for local archive questions.

## Data Scope

This release syncs the Google Health data types exposed to the authorized
account, including activity, sleep, heart, HRV, SpO2, respiratory rate,
temperature derivations, body metrics, hydration, profile, settings, and
observed device/source context.

## Upgrade And Migration Notes

- The module path is `github.com/fdsouvenir/ghcli`.
- Default local state is stored under `~/.local/state/ghcli` unless
  `$XDG_STATE_HOME` or `--store` is set.
- The default KeePassXC token entry is `Services/google-health-ghcli`.
- `ghcli` uses the Google Health API for the same underlying health history.
